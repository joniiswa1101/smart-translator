Langkah 2 — Buat middleware enforceLicense
File baru: artifacts/api-server/src/middleware/enforceLicense.ts
Logic:
Ambil deviceToken dari header Authorization: Bearer <token> (bukan header custom mentah seperti x-device-id — harus signed JWT, lihat Langkah 3).
Verify JWT signature. Kalau invalid/expired → 401.
Decode deviceId dari token, query row noturaTrialDevices.
Kalau device tidak ditemukan → 403.
Cek apakah cycleStartDate sudah lewat 30 hari dari sekarang:
Kalau ya: reset meetingsThisMonth = 0, cloudMinutesUsedThisMonth = 0, cycleStartDate = now().
Kalau tier === 'free':
Kalau meetingsThisMonth >= 5 → 402 { error: 'free_tier_limit', upgrade: true }
Kalau durasi recording yang diminta (dari req.body.durationSeconds) > 1800 detik → 402 { error: 'free_tier_duration_cap', upgrade: true }
Kalau tier === 'pro':
Kalau cloudMinutesUsedThisMonth >= FAIR_USE_CAP_MINUTES (definisikan konstanta, default 600 menit/bulan, boleh disesuaikan) → 429 { error: 'fair_use_limit' }
Lolos semua cek → attach req.deviceId dan req.deviceTier, lanjut next().
Langkah 3 — Device authentication dengan JWT (CRITICAL — jangan skip)
Header device id mentah BISA dipalsukan oleh client. Harus pakai signed token:
Saat device pertama kali register (cari atau buat endpoint registrasi device, kemungkinan sudah ada di trial.ts), server generate JWT yang isinya { deviceId }, signed dengan secret dari environment variable (JWT_SECRET — cek apakah sudah ada di Replit Secrets, kalau belum, buat dan beri tahu saya nama variable-nya).
Token ini dikirim balik ke client, client simpan di secure storage (Expo SecureStore), dan dikirim di setiap request sebagai Authorization: Bearer <token>.
Token TIDAK boleh expired terlalu cepat (set expiry panjang, misal 1 tahun, karena ini bukan auth session — ini device identity), tapi harus bisa di-revoke kalau device di-reset/uninstall (opsional: tambah kolom tokenRevoked boolean untuk kill switch manual).
