# B6.5: Perbandingan Solusi A vs B

## Ringkasan Executive

| Aspek | Solusi A (Multi-input + Single Output) | Solusi B (Multi-input + Multi-output) |
|-------|----------------------------------------|----------------------------------------|
| **Arsitektur** | Audio proxy ke OpenAI Realtime (gpt-realtime-translate) | Pivot-teks: ASR → Translate → TTS |
| **Input** | Multi client (mic) | Multi client (mic) |
| **Output** | Single speaker (broadcast) | Individual (per peserta, berbeda bahasa) |
| **Bahasa** | 2 bahasa (ID ↔ EN) | 3 bahasa (ID, EN, BN) |
| **Audio Quality** | 24kHz PCM, OpenAI native | 24kHz PCM, tts-1 |
| **Latency** | ~800-1200ms (realtime) | ~1500-4000ms (tergantung panjang) |
| **Biaya API** | Tinggi (gpt-realtime) | Lebih rendah (whisper-1 + gpt-4o-mini + tts-1) |
| **Routing** | 1 target (semua dengar sama) | Multi-target (tiap peserta dengar bahasa-nya) |
| **Keterbatasan** | Hanya 2 bahasa, tidak bisa BN | Semua bahasa didukung |
| **Rekomendasi** | Legacy | **Produksi** |

---

## Detail Latency

### Solusi A (gpt-realtime-translate)

| Tahap | Estimasi | Catatan |
|-------|----------|---------|
| Audio capture → VAD | 0-200ms | Client-side VAD |
| VAD silence → commit | 300-500ms | Silence timer |
| OpenAI first byte | 200-500ms | Streaming audio |
| Total dead-air | **800-1200ms** | Untuk frase pendek |
| End-to-end | 1200-2500ms | Tergantung panjang ucapan |

### Solusi B (Pivot-Teks)

| Tahap | Estimasi | Catatan |
|-------|----------|---------|
| Audio capture → VAD | 0-200ms | Client-side VAD |
| VAD silence → commit | 300-500ms | Silence timer |
| ASR (whisper-1) | 300-800ms | 1-3 detik audio |
| Translate (gpt-4o-mini) | 200-500ms | Paralel per target |
| TTS (tts-1) | 500-1500ms | Paralel per target |
| Chunked fan-out | 100-200ms | Streaming ke client |
| Total dead-air | **1500-4000ms** | Tergantung panjang ucapan |

**Catatan:** Solusi B lebih lambat karena ada 3 tahap serial (ASR → Translate → TTS) vs Solusi A yang 1 pipeline streaming. Tetapi Solusi B jauh lebih fleksibel.

## Detail Biaya API

### Solusi A (per jam)

- gpt-realtime-translate: ~$0.006-0.012/minute audio
- Estimasi untuk 1 jam training: ~$3-7

### Solusi B (per jam)

- whisper-1: ~$0.006/minute audio
- gpt-4o-mini: ~$0.015/1K tokens (translate)
- tts-1: ~$0.015/1K characters (TTS)
- Estimasi untuk 1 jam training: ~$1-3

**Solusi B ~60% lebih murah** untuk usage tinggi.

## Keputusan B6.7

### Rekomendasi: **Solusi B untuk Produksi**

**Alasan:**
1. **Multi-bahasa**: Mendukung 3 bahasa (ID, EN, BN) — penting untuk training dengan peserta Bengali
2. **Individual routing**: Tiap peserta dengar bahasa-nya sendiri — lebih fleksibel
3. **Biaya lebih rendah**: ~60% lebih murah untuk usage tinggi
4. **Fleksibilitas**: Bisa menambah bahasa tanpa ganti arsitektur
5. **Glosarium**: Dukungan domain-specific terms untuk akurasi lebih baik

**Kompromi:**
- Latency lebih tinggi (~1.5-4s vs ~0.8-1.2s)
- Butuh 3 API call vs 1 streaming call
- TTS quality mungkin sedikit di bawah gpt-realtime

**Rekomendasi lanjutan:**
- Untuk demo/POC: Solusi A (lebih cepat, lebih simpel)
- Untuk produksi: Solusi B (lebih fleksibel, lebih murah)
- Untuk latency kritikal: Optimasi B (streaming TTS, overlap ASR+translate)
